Question 1 : Why is StringBuilder preferred over string concatenation (+=) inside loops?
Answer: Because in java String can never be changed after created , so typing += will create new object and
its a big problem when doing this inside a loop which will slow the application.

Question 2 : For each method below, explain which collection you selected and why:
             getAllWords()
             getUniqueWords()
             getWordFrequency()
             containsBothWords()
Answer:
getAllWords()=> List selected because we need to keep duplicates and keep the order.
getUniqueWords()=> HashSet selected because order does not matter and duplicates are not allowed.
getWordFrequency()=> HashMap selected because we need key,value so Map is the only collection that supports this
key-value relationship directly.
containsBothWords()=> HashSet selected because order does not matter what does matter is existence only.
